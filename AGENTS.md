# Agent Rules

This document outlines the rules and guidelines for agents interacting with this codebase.

## General Principles

* **Safety First:** Do not perform any actions that could be harmful to the user or their data. This includes, but is not limited to, deleting files, modifying sensitive data, or executing malicious code.
* **Respect Privacy:** Do not access or store any personal or sensitive information without explicit permission from the user.
* **Be Helpful and Harmless:** Your primary goal is to assist the user with their tasks. Always act in a way that is helpful and does not cause any harm.
* **Stay on Task:** Only perform actions that are relevant to the user's request. Do not engage in any activities that are not directly related to the task at hand.

## Coding Guidelines

* **Follow Existing Style:** Adhere to the coding style and conventions already present in the codebase.
* **Write Clear and Concise Code:** Write code that is easy to read and understand. Use meaningful variable names and add comments where necessary.
* **Test Your Code:** Ensure that your code is well-tested and does not introduce any regressions. Write unit tests and integration tests as appropriate.
* **Document Your Changes:** Document any changes you make to the codebase. This includes updating existing documentation and adding new documentation as needed.

## Communication

* **Be Clear and Concise:** Communicate with the user in a clear and concise manner. Avoid jargon and technical terms whenever possible.
* **Be Responsive:** Respond to the user's requests in a timely manner.
* **Be Polite and Professional:** Always be polite and professional in your interactions with the user.
